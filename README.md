# chrome_fix
🚀 Быстрый способ починить доступ к легитимным сайтам, заблокированным ТСПУ РКН, без VPN и прокси. 

**[UPD: Если Вы владелец сайта - Как починить блокировку Вашего сайта от РКН ТСПУ — реальный кейс](https://habr.com/ru/articles/1045684/)**
-
[**Как починить блокировку легальных сайтов РКН ТСПУ одной строчкой в Chrome**](https://habr.com/ru/articles/1045438/)

У многих сейчас бесконечно грузятся или выдают ошибку (ERR_CONNECTION_CLOSED) нормальные сайты — например, хостинг beget.com, GitHub, Discord и другие ресурсы. Причина — «умные» блокировки ТСПУ, которые триггерятся на TLS-отпечатки (JA3/JA4) браузеров. 

Есть простое решение для ПК и смартфонов на базе Chromium (Google Chrome, Яндекс.Браузер, Edge, Opera). Изменение одного флага полностью меняет порядок шифров в TLS Client Hello, и ТСПУ перестает блокировать пакеты.

🛠 ИНСТРУКЦИЯ (ВРУЧНУЮ):
1. Вставьте в строку браузера: chrome://flags/
2. Найдите через поиск: Cryptography Compliance (CNSA) (#cryptography-compliance-cnsa)
3. Переключите его в режим: Enabled
4. Нажмите кнопку Relaunch (Перезагрузить) внизу экрана.


⚡️ АВТОМАТИЗАЦИЯ В 1 КЛИК ДЛЯ WINDOWS (для ленивых или друзей):
Чтобы не лазить по настройкам, можно применить этот флаг автоматически через реестр одним кликом:
👉 Скачайте `.bat` скрипт: https://github.com/ebyebots/chrome_fix/blob/main/chrome_fix.bat
(Если не применилось сразу, запустите файл от имени Администратора и перезапустите браузер).

Включение Enabe becnhmarking

Устанавливает для всех функций фиксированное состояние, то есть отключает рандомизацию состояний функций. Если выбран параметр «(Состояния функций по умолчанию)», все функции устанавливаются в состояние по умолчанию. Если выбран параметр «(Соответствие конфигурации полевых испытаний)», все функции устанавливаются в состояние, заданное в конфигурации полевых испытаний. Это используется разработчиками и тестировщиками для диагностики того, вызвана ли наблюдаемая проблема нестандартной конфигурацией base::Feature. Этот флаг автоматически сбрасывается после 3 перезапусков и будет выключен с 4-го перезапуска. При 3-м перезапуске флаг будет казаться выключенным, но эффект все еще будет действовать. – Mac, Windows, Linux, ChromeOS, Android
